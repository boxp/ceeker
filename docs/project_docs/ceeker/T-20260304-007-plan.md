# T-20260304-007: カード表示の複数行メッセージ崩れ修正

## 問題

ceekerのカード表示モードで、最終メッセージが長文（特に日本語/全角文字混在）の場合、
左枠（`│`）を本文が貫通してレイアウトが崩れる。

### 原因

1. `truncate`関数が`count`（文字数）で切り詰めており、CJK全角文字の表示幅（2カラム）を考慮していない
2. メッセージに改行が含まれる場合のハンドリングがない
3. 折り返し時に2行目以降に`│`プレフィックスが付与されない

## 修正内容

### 1. 表示幅計算関数の追加 (`view.clj`)

- `char-display-width`: Unicode範囲に基づき文字の表示幅を返す（CJK=2, その他=1）
- `str-display-width`: 文字列全体の表示幅を計算
- `substr-by-width`: 表示幅ベースで最長プレフィックスを抽出
- `truncate-by-width`: 表示幅ベースで切り詰め＋省略記号
- `wrap-by-width`: 表示幅ベースで複数行に折り返し

### 2. カードメッセージ描画の修正

- `card-message-lines`: メッセージを表示幅ベースでラップし、各行に`│`プレフィックスを付与
  - 改行文字はスペースに正規化
  - 最大行数（3行）を超える場合は省略記号で打ち切り
- `format-session-card`: 単一行の`line3`から`card-message-lines`による複数行対応に変更
- `card-line2`: cwd名の切り詰めも`truncate-by-width`に変更

### 3. 対象範囲

- カード表示のメッセージ行のみ修正
- テーブル表示モードは非スコープ（`format-session-line`は変更なし）
